## 关键数据要点（演讲参考）

### 可以口头提到的数据：

1. **79个问题** - 不是简单的事实查询，而是需要推理的"谜题"
2. **平均6-8个约束** - 每个问题都需要同时满足多个条件
3. **3-7跳推理** - 需要在知识图谱中进行多次跳转
4. **60%问题找论文** - 最常见的答案是论文标题
5. **跨时间跨度** - 从1810年代到2023年，跨度超过200年

### 可以强调的挑战点：

1. **"谜题式"设计** - 问题不是直接问，而是通过约束间接描述
2. **外部知识依赖** - 需要知道大学成立时间、历史事件等
3. **跨文档关联** - 需要同时查看多篇论文和作者信息
4. **精确属性要求** - "8个词"、"27条引用"等精确数字

## 设计建议

### 配色方案

```
背景色: #FFFFFF (白色) 或 #F5F5F5 (浅灰)
标题色: #2C3E50 (深蓝灰)
正文色: #34495E (灰蓝)
强调色: #3498DB (蓝色) 或 #E74C3C (红色)
图表色: 使用5种节点类型的对应颜色
  Paper: #FF6B6B (红)
  Author: #4ECDC4 (青)
  Institution: #45B7D1 (蓝)
  Venue: #96CEB4 (绿)
  Entity: #FFEAA7 (黄)
```

### 字体建议

```
标题: Microsoft YaHei / SimHei, 32-36pt, 加粗
正文: Microsoft YaHei, 18-24pt, 常规
代码/示例: Consolas / Courier New, 14-16pt
数字: Arial, 24-32pt
```
